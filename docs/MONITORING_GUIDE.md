## All Prometheus Metrics (28 Metrics)

| `predictions_total` | Counter | `problem_statement`, `risk_tier` | Total predictions by PS and risk tier |

## Best Grafana Dashboards to Build

### Dashboard 1: Operations Overview (must-have)

| Panel | PromQL Query | Visualization | Why |
|-------|-------------|---------------|-----|
| **Predictions/min** | `rate(predictions_total[1m]) * 60` | Time series (line) | Traffic volume — is the API being used? |
| **Error Rate %** | `rate(errors_total[5m]) / rate(http_requests_total[5m]) * 100` | Time series + threshold | Service health — are errors spiking? |
| **P50/P95/P99 Latency** | `histogram_quantile(0.99, rate(prediction_latency_seconds_bucket[5m])) * 1000` | Time series (multi-line) | Performance — is the API fast enough? |
| **Requests In Progress** | `requests_in_progress` | Gauge | Load — is the API overloaded? |
| **Uptime** | `time() - process_start_time_seconds` | Stat | Reliability — when did API last restart? |
| **Memory Usage** | `process_resident_memory_bytes / 1024 / 1024` | Gauge (MB) | Resource — is it leaking memory? |

### Dashboard 2: Model Performance (critical for ML)

| Panel | PromQL Query | Visualization | Why |
|-------|-------------|---------------|-----|
| **Failure Probability Distribution** | `histogram_quantile(0.5, failure_probability_distribution_bucket)` | Heatmap / Histogram | Is the model confident or uncertain? |
| **Anomaly Score Distribution** | `anomaly_score_distribution_bucket` | Histogram | Are anomaly scores clustering near threshold? |
| **SLA Risk Score Distribution** | `sla_risk_score_distribution_bucket` | Histogram | Are most devices low or high risk? |
| **Risk Tier Breakdown** | `risk_tier_total` | Pie chart | What % of predictions are CRITICAL vs LOW? |
| **Mean Health Score** | `mean_health_score` | Gauge (0-100) | Fleet-wide health trending up or down? |
| **Failure Prob Trend** | `failure_probability_last` | Time series | Is the latest prediction healthy or concerning? |

### Dashboard 3: SLA & Reliability (business-facing)

| Panel | PromQL Query | Visualization | Why |
|-------|-------------|---------------|-----|
| **Devices At Risk** | `devices_at_risk` | Stat (big number, red) | How many need immediate attention? |
| **SLA Breach Probability** | `sla_breach_probability_avg` | Gauge (0-1) | Likelihood of SLA violation |
| **RUL Distribution** | `rul_estimate_days_distribution_bucket` | Bar chart | How many devices have <14 days left? |
| **CRITICAL Predictions/hr** | `rate(risk_tier_total{tier="CRITICAL"}[1h]) * 3600` | Time series | Are critical alerts increasing? |
| **Predictions by PS** | `sum by(problem_statement)(predictions_total)` | Bar chart | Which PS is most used? |

### Dashboard 4: Data Quality & Drift (MLOps)

| Panel | PromQL Query | Visualization | Why |
|-------|-------------|---------------|-----|
| **Drift Share** | `drift_share_current * 100` | Gauge (0-100%, red >30%) | Is data drifting? Do we need to retrain? |
| **Data Quality Pass Rate** | `data_quality_pass_rate` | Gauge (0-100%) | Are quality gates passing? |
| **Retraining Count** | `retraining_triggered_total` | Stat | How often has retraining been triggered? |
| **Models Loaded** | `model_loaded` | Table | Which models are active? |

## Grafana Alert Rules

| Alert | Condition | Severity | Action |
|-------|-----------|----------|--------|
| **High Error Rate** | `rate(errors_total[5m]) > 0.1` | Critical | Page on-call, check API logs |
| **High Latency** | `histogram_quantile(0.99, ...) > 1.0` | Warning | Scale up replicas |
| **Drift Detected** | `drift_share_current > 0.30` | Warning | Trigger retraining |
| **Quality Degraded** | `data_quality_pass_rate < 90` | Warning | Check data pipeline |
| **Many Critical Devices** | `devices_at_risk > 20` | Critical | Alert maintenance team |
| **Model Not Loaded** | `model_loaded == 0` | Critical | API restart needed |
| **High Memory** | `process_resident_memory_bytes > 2e9` | Warning | Memory leak investigation |

## Setting Up Grafana Dashboards

### Step 1: Start the stack

```powershell
cd monitoring
docker compose up -d
```

### Step 2: Login to Grafana

Open http://localhost:3000 → Login: `admin` / `admin`

### Step 3: Add Prometheus data source

1. Go to **Connections** → **Data Sources** → **Add data source**
2. Select **Prometheus**
3. URL: `http://prometheus:9090` (if in Docker) or `http://host.docker.internal:9090`
4. Click **Save & Test**

### Step 4: Create dashboard

1. Click **+** → **New Dashboard** → **Add Visualization**
2. Select Prometheus data source
3. Enter PromQL query from the tables above
4. Choose visualization type
5. Set thresholds (green/yellow/red)

## Azure Application Insights

### How to Set Up

#### 1. Install the SDK

```powershell
pip install opencensus-ext-azure opencensus-ext-fastapi
```

#### 2. Add to FastAPI (api/main.py)

```python
import os
from opencensus.ext.azure.trace_exporter import AzureExporter
from opencensus.ext.azure.log_exporter import AzureLogHandler
from opencensus.trace.samplers import ProbabilitySampler
import logging

# Connection string from Azure Portal → App Insights → Overview
APPINSIGHTS_CONNECTION = os.getenv(
    "APPLICATIONINSIGHTS_CONNECTION_STRING",
    "InstrumentationKey=your-key-here"
)

# Traces (request tracking)
from opencensus.ext.fastapi.fastapi_middleware import FastAPIMiddleware
app.add_middleware(
    FastAPIMiddleware,
    exporter=AzureExporter(connection_string=APPINSIGHTS_CONNECTION),
    sampler=ProbabilitySampler(rate=1.0)   # 1.0 = 100% sampling
)

# Logs
logger = logging.getLogger(__name__)
logger.addHandler(AzureLogHandler(connection_string=APPINSIGHTS_CONNECTION))

# Custom metrics (equivalent to Prometheus counters/gauges)
from opencensus.ext.azure import metrics_exporter
exporter = metrics_exporter.new_metrics_exporter(
    connection_string=APPINSIGHTS_CONNECTION
)
```

#### 3. Log custom metrics

```python
from opencensus.stats import aggregation, measure, stats, view

# Define measures
failure_prob_measure = measure.MeasureFloat("failure_probability", "Failure prob", "prob")
latency_measure = measure.MeasureFloat("prediction_latency", "Prediction latency", "ms")
risk_tier_measure = measure.MeasureInt("risk_tier_count", "Risk tier count", "count")

# Define views (how to aggregate)
failure_prob_view = view.View(
    "failure_probability_distribution",
    "Distribution of failure probabilities",
    [],
    failure_prob_measure,
    aggregation.DistributionAggregation([0.1, 0.2, 0.3, 0.5, 0.7, 0.9])
)

# Register and record
stats.stats.view_manager.register_view(failure_prob_view)
mmap = stats.stats.stats_recorder.new_measurement_map()
mmap.measure_float_put(failure_prob_measure, prob)
mmap.record()
```

### Querying in Azure App Insights (KQL)

Open Azure Portal → Application Insights → Logs

#### Request Metrics

```kusto
// Predictions per minute
requests
| where name contains "predict"
| summarize count() by bin(timestamp, 1m)
| render timechart

// P50, P95, P99 latency
requests
| where name contains "predict"
| summarize
    p50 = percentile(duration, 50),
    p95 = percentile(duration, 95),
    p99 = percentile(duration, 99)
  by bin(timestamp, 5m)
| render timechart

// Error rate by endpoint
requests
| where timestamp > ago(1h)
| summarize
    total = count(),
    failed = countif(success == false)
  by name
| extend error_rate = round(todouble(failed) / total * 100, 2)
| order by error_rate desc

// Slowest endpoints
requests
| where timestamp > ago(1h)
| summarize avg_duration = avg(duration), p99 = percentile(duration, 99) by name
| order by p99 desc
```

#### Model Performance (Custom Metrics)

```kusto
// Failure probability distribution
customMetrics
| where name == "failure_probability"
| summarize avg(value), percentile(value, 95) by bin(timestamp, 5m)
| render timechart

// Risk tier breakdown
customMetrics
| where name == "risk_tier_count"
| extend tier = tostring(customDimensions["tier"])
| summarize count() by tier
| render piechart

// SLA risk score trend
customMetrics
| where name == "sla_risk_score"
| summarize avg(value), max(value) by bin(timestamp, 1h)
| render timechart

// Anomaly detection rate
customMetrics
| where name == "anomaly_detected"
| summarize anomalies = countif(value == 1), total = count() by bin(timestamp, 1h)
| extend rate = round(todouble(anomalies) / total * 100, 2)
| render timechart
```

#### Drift & Quality

```kusto
// Data drift trend
customMetrics
| where name == "drift_share"
| project timestamp, drift_pct = value * 100
| render timechart

// Data quality trend
customMetrics
| where name == "data_quality_pass_rate"
| render timechart

// Retraining events
customEvents
| where name == "model_retrained"
| project timestamp, reason = tostring(customDimensions["reason"])
```

#### System Health

```kusto
// API availability
availabilityResults
| summarize availability = avg(success) * 100 by bin(timestamp, 1h)
| render timechart

// Exceptions
exceptions
| where timestamp > ago(24h)
| summarize count() by type, bin(timestamp, 1h)
| render timechart

// Dependencies (external calls)
dependencies
| where timestamp > ago(1h)
| summarize avg(duration), count() by name
| order by count_ desc
```

### App Insights Alerts (equivalent to Grafana alerts)

Set up in Azure Portal → Application Insights → Alerts → New Alert Rule:

| Alert | KQL Condition | Severity |
|-------|--------------|----------|
| High Error Rate | `requests \| where success == false \| count > 10` (per 5 min) | Sev 1 |
| High Latency | `requests \| percentile(duration, 99) > 1000` | Sev 2 |
| Model Drift | `customMetrics \| where name == "drift_share" and value > 0.30` | Sev 2 |
| Critical Devices | `customMetrics \| where name == "devices_at_risk" and value > 20` | Sev 1 |
| API Down | `availabilityResults \| avg(success) < 0.95` | Sev 0 |

## Prometheus vs Azure App Insights — When to Use

| Aspect | Prometheus + Grafana | Azure App Insights |
|--------|---------------------|-------------------|
| **Best for** | Local dev, Docker, self-hosted | Azure production |
| **Cost** | Free (self-hosted) | Pay per GB ingested (~$2.30/GB) |
| **Setup** | Docker Compose (5 min) | Azure resource + SDK (15 min) |
| **Dashboards** | Grafana (highly customizable) | Azure Portal (integrated) |
| **Alerting** | Grafana alerts | Azure Monitor alerts (SMS, email, webhook) |
| **Retention** | Configurable (default 30d) | 90 days (configurable to 730d) |
| **Distributed tracing** | Needs Jaeger/Zipkin | Built-in |
| **Log correlation** | Manual | Automatic (request → logs → exceptions) |
| **Recommendation** | Use in development | Use in Azure production |
